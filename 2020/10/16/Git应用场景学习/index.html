Git应用场景学习

Git应用场景学习

  • 这部分blog记录一些Git常用的应用场景以及相关的命令，持续更新。

远程创建分支，关联本地分支到远程分支

  • git remote add remote_git_name remote_git_address
    • 添加远程库，为远程库命名
  • git branch -d local_branch_name
    • 创建并切换至本地分支
  • git push remote_git_name local_branch_name:remote_branch_name
    • 将本地分支推送至远程分支上，如果没有指定remote_branch_name则推送该分支默认关联的分支
    • 如果在git上不存在名为remote_branch_name的分支，则自动创建名为remote_branch_name的分支
  • git branch --set-upstream-to=remote_git_name/remote_branch_name
    • 将当前分支关联到远程分支上，如此在这个分支下的git push和pull都能直接推送到这个分支上
  • git branch -a
    • 查看远程分支

git如何修改已提交的commit

本地修改-修改最近一次

  • git commit --amend
    • 修改最近一次的commmit
    • 使用后会进入最近的一个文本编辑器可以修改描述内容
  • git add <filename> 
git commit --amend
    • 修改提交的文件

本地修改-修改任意提交历史位置Commit

  • 参考URL https://github.com/uolcano/blog/issues/12
    • 修改commit
    • 本地回退等
  • 可以通过变基命令，修改最近一次提交以前的某次提交。不过修改的提交到当前提交之间的所有提交的hash值都会改变。
    变基操作需要非常小心，一定要多用git status命令来查看你是否还处于变基操作，可能某次误操作的会对后面的提交历史造成很大影响。
  • git log
  • git rebase -i <commit range>
    • 注意:变基命令打开的文本编辑器中的commit顺序跟git log查看的顺序是相反的，也就是最近的提交在下面，老旧的提交在上面
    • 注意:变基命令其实可以同时对多个提交进行修改，只需要修改将对应行前的pick都修改为edit，保存退出后会根据你修改的数目多次打开修改某次commit的文本编辑器界面。但是这个范围内的最终祖先commit不能修改，也就是如果有5行commit信息，你只能修改下面4行的，这不仅限于commit修改，重排、删除以及合并都如此。
  • git commit --amend
    • 提交
  • git rebase --continue
    • 完成变基操作

远程修改

  • git push -f <remote> <branch>
